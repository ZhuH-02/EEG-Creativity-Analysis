# EEG Creativity Analysis (EEG → Features → Baseline ML)

This repository contains a reproducible starter pipeline for feature extraction and baseline machine-learning experiments on an open EEG creativity dataset.

## Dataset (high level)

The project targets classification/analysis of cognitive phases in a design creativity micro-cycle:

- Rest (**RST**)
- Idea Generation (**IDG**)
- Idea Evolution (**IDE**)
- Idea Rating (**IDR**)

Dataset description used for this project: preprocessed and segmented EEG from **28** right-handed graduate engineering students (4 female), recorded with a **64-channel BrainVision actiCHamp** system at **500 Hz**, and segmented into phase-labeled epochs (see DOI documentation for full details).

Public dataset DOI: `10.17632/24yp3xp58b.1`

## Configuration (no hardcoded paths)

Project paths are derived relative to the repository root inside [code/config.py](code/config.py).

Key settings you may adjust:

- `DATA_DIR` (defaults to `./EEG data`)
- `SAMPLING_RATE` (defaults to 500)
- `WINDOW_SIZE`, `WINDOW_OVERLAP`
- Output locations: `RESULTS_DIR`, `MODELS_DIR`

## Reproducible Phase 1: data audit + feature table

This script validates the local files, extracts window-level EEG features, and generates a small set of summary tables and readable plots.

```

Outputs are written under `./outputs/phase1_data_selection/`.

## Baseline model run (saves a small output bundle per run)

```powershell

```

Each run creates a timestamped folder under `./results/week1_baseline/` and saves:

- Metrics (JSON)
- Classification report (text)
- Confusion matrix plot
- ROC curve plot (binary classification only)

- Participant ID should not be used as a model feature. Use subject-wise splitting (e.g., leave-one-subject-out) for realistic evaluation.

## Troubleshooting

### Issue: "Module not found" errors

```bash
# Reinstall requirements
pip install -r requirements.txt --upgrade
```

### Issue: Data file not found

- Check file paths in your code match the actual directory structure
- Ensure .eeg, .vhdr, and .vmrk files are in the same directory
- Verify the participant ID matches the folder name

### Issue: Memory errors with large datasets

```python
# Use windowing for longer signals
features = extractor.extract_batch_features(
    signal_data, 
    window_size=1000  # Adjust based on available memory
)
```

### Issue: Low accuracy in Week 1

- This is **expected** - baseline should be 55-70%
- More sophisticated features in Week 2 will improve performance
- Ensure data labels are correct
- Check for data quality issues (artifacts, noise)
